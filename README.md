# 🛡️ VerifyMed 

**VerifyMed** is an AI-powered, real-time drug authentication engine designed to combat counterfeit medications in Sub-Saharan Africa. It uses a swarm of 5 specialized AI agents to analyze a drug's identity, search global registries, cross-reference recall alerts, and verify pharmacological consistency—all in under 30 seconds.

![VerifyMed Banner](https://i.imgur.com/G9L2rUe.png) *(Placeholder for app screenshot)*

## ✨ Features

- 📸 **Live Barcode Scanner**: Mandates scanning of the drug packaging (1D/2D DataMatrix) to ensure high-accuracy data extraction.
- 💬 **Conversational AI Intake**: A smart chat assistant gathers patient symptoms and details of all drugs being taken.
- 🤖 **Multi-Agent Orchestration**:
  - **Stage 0**: Groq Vision OCR extracts text from packaging if the user uploads an image.
  - **Agent 1 (Identity)**: Searches NAFDAC and WHO registries to verify the manufacturer.
  - **Agent 2 (Intelligence)**: Scours the global web for active counterfeit warnings or recalls.
  - **Agent 3 (Science)**: Checks if the active ingredients actually match the claimed therapeutic use.
  - **Agent 4 (Pharmacovigilance)**: Queries the US Government OpenFDA database for drug-drug interactions and severe side effects.
  - **Agent 5 (Safety Advisor)**: Synthesizes all data against the patient's symptoms to output a final strict medical-legal verdict.
- 📱 **Mobile Authentication Service (MAS) Integration**: Explicitly guides the user to scratch the MAS panel and verifies it via a one-click SMS button.

## 🚀 Running Locally

### 1. Backend Setup
The backend runs on **FastAPI** and uses **Groq** for high-speed AI inference.

1. Install the dependencies:
   ```bash
   pip install -r requirements.txt
   ```
2. Create a `.env` file in the root directory and add your Groq API key:
   ```env
   GROQ_API_KEY=your_groq_api_key_here
   ```
3. Start the FastAPI server:
   ```bash
   uvicorn main:app --port 8000
   ```

### 2. Frontend Setup
Because the backend is configured to serve static files from the root, you can view the app immediately by navigating to:
👉 `http://localhost:8000/static/index.html`

## 🌍 Deployment Guide

### Hosting the Frontend (Netlify)
Netlify is perfect for hosting the `index.html`, `styles.css`, and `app.js` files.

1. Create an account on [Netlify](https://www.netlify.com/).
2. You can either link this GitHub repository and configure the publish directory to `/`, **OR** you can simply drag and drop the folder containing `index.html`, `styles.css`, and `app.js` directly into the Netlify dashboard.
3. **IMPORTANT**: Before deploying the frontend to production, you must update the `API_BASE` variable in `app.js` to point to your live backend URL instead of `http://localhost:8000`.

### Hosting the Backend (Render / Railway / Heroku)
Because the backend is a Python FastAPI application, it needs to be hosted on a platform that supports Python servers (Netlify only supports static files and serverless functions).

1. Push this repository to GitHub.
2. Connect the repository to [Render](https://render.com/) or [Railway](https://railway.app/).
3. Set the start command to:
   ```bash
   uvicorn main:app --host 0.0.0.0 --port $PORT
   ```
4. Add your `GROQ_API_KEY` to the environment variables in the deployment dashboard.
5. Take the public URL generated by Render/Railway and paste it into `app.js` on the frontend as the `API_BASE`.
